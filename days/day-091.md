# Ansible Lineinfile Module

The Nautilus DevOps team needs to automate the installation and configuration of a simple **Apache (`httpd`) web server** across all application servers in Stratos DC. Along with the setup, a sample web page must be deployed and modified using Ansible modules while ensuring proper ownership and permissions.

We already have an inventory file under `/home/thor/ansible` on the jump host. The goal is to create a playbook `playbook.yml` in the same directory and execute all required tasks using Ansible.

---

##  Steps to Execute

### 1. Navigate to Ansible directory

```bash
cd /home/thor/ansible
```

### 2. Verify inventory file

```bash
cat inventory
```

### 3. Create playbook file

```bash
touch playbook.yml
```

### 4. Add Playbook Content

Paste the following configuration  [`PLAYBOOK`](../files/ansible_playbook_lineinfile_module_d91.yml).

### 5. Run the Playbook

```bash
ansible-playbook -i inventory playbook.yml
```

### 6. Verify Deployment

```bash
curl http://stapp02

```

##  Good to Know

* **Ansible `lineinfile` module**
  Used to ensure a specific line exists in a file. It avoids duplication and maintains idempotency.

* **`insertbefore: BOF`**
  BOF = *Beginning Of File* → ensures the line is inserted at the top.

* **`copy` module behavior**
  Overwrites the file if it exists. Ideal for setting a **base state**.

* **File permissions `0755`**

  * Owner: read, write, execute
  * Group/Others: read, execute

* **Apache (`httpd`) service**
  Must be **started and enabled** to serve web content after reboot.

* **Idempotency principle **
  Running the playbook multiple times will not change the result unnecessarily.

* **Task execution order matters ⚙️**

  * First: install package
  * Then: create file
  * Then: modify file
  * Finally: ensure service is running

* **No restart required for HTML changes **
  Apache serves static files directly — no service restart needed for simple content updates.
